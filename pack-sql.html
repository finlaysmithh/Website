SQL & Big Data Analysis

Goal (≈10 hours)

Design a relational database schema, practice powerful SQL queries and analyse a large public dataset to draw meaningful insights while learning optimisation techniques.

Suggested tools:

  • SQLite/MySQL/PostgreSQL
  • Database IDE (DB Browser, pgAdmin)
  • Kaggle or public datasets
  • ChatGPT

Guided tasks

  1. Design a relational schema. Choose a domain (e.g., retail, HR) and create tables with primary and foreign keys to store realistic information. Draw an entity‑relationship diagram to depict your design and briefly describe your normalization strategy. Tip: Normalise to reduce redundancy; ask ChatGPT if you need help understanding normal forms.
  2. Populate and query your database. Insert at least 20–50 rows per table. Write SQL queries to answer business questions: top‑selling products, average spend per customer, order frequencies. Use JOIN, GROUP BY, HAVING, window functions and subqueries. Tip: Use your database IDE to test queries and experiment with CTEs for readability.
  3. Analyse a large dataset. Download a sizeable dataset (e.g., from Kaggle) and load it into your database. Write complex queries to compute summary statistics, identify trends and correlations. Try combining this dataset with your own tables if relevant. Optimise your queries using indexes and compare execution plans with EXPLAIN. Tip: ChatGPT can suggest indexing strategies or query rewrites.
  4. Summarise your work. Prepare a short report documenting your schema, key queries, optimisation steps and insights. Include screenshots or exported query results. Reflect on the challenges and lessons learned in database design and big data analysis. Tip: Use clear headings and bullet points to organise your report.

Tip: Use available tutorials, sample projects or ask ChatGPT for guidance as you complete each task.

Need help getting started? Break each task down into small steps, explore free tutorials online and ask ChatGPT to explain unfamiliar concepts. Practise iteratively and document your progress as you go.

What to submit

  • Your schema (DDL script or ER diagram)
  • SQL scripts for your queries
  • Report with findings and screenshots
