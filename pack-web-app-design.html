Web & App Design (UX/UI)

Goal (≈10 hours)

Learn to design user‑friendly digital experiences by taking a concept through wireframing, prototyping and implementation. This pack encourages hands‑on practice with both design tools and front‑end code.

Suggested tools:

  • Figma, Adobe XD or Sketch
  • HTML, CSS & JavaScript
  • Accessibility checkers
  • Usability testing with friends/family
  • ChatGPT

Guided tasks

  1. Define the problem & sketch user flows. Identify a simple app idea (e.g., budget tracker, recipe finder). Write a brief problem statement and describe the primary user persona. Sketch user flows and low‑fidelity wireframes for key screens using paper or digital tools. Tip: Focus on the user’s goals and critical tasks when designing flows.
  2. Create a high‑fidelity prototype. Use Figma, XD or Sketch to translate your wireframes into a polished prototype. Choose a harmonious colour palette, accessible typography and consistent spacing. Add interactive hotspots to simulate navigation. Test your prototype by clicking through it yourself. Tip: Use built‑in accessibility plugins to check colour contrast and font sizes.
  3. Build a responsive website. Implement one or two pages of your prototype using semantic HTML, modern CSS (flexbox/grid) and a touch of JavaScript for interactivity (e.g., collapsible menus, input validation). Ensure the layout adapts gracefully across screen sizes. Comment your code to explain major sections. Tip: Use browser developer tools to debug layout issues and test on mobile screens.
  4. Gather feedback & iterate. Ask at least two people to test your prototype or website. Provide them with tasks to complete and ask for candid feedback on usability and aesthetics. Summarise their comments and outline specific changes you would make to improve the design. Tip: Observe testers silently so you can see where they get confused without influencing them.

Tip: Use available tutorials, sample projects or ask ChatGPT for guidance as you complete each task.

Need help getting started? Break each task down into small steps, explore free tutorials online and ask ChatGPT to explain unfamiliar concepts. Practise iteratively and document your progress as you go.

What to submit

  • Your problem statement, user persona and wireframes
  • A link or file to your high‑fidelity prototype
  • Source code for your responsive website
  • A feedback summary and list of planned improvements

Send your deliverables to Finlay@experience4grads.co.uk. Your submission will be reviewed within 48 hours. If approved, you’ll receive a certificate and LinkedIn endorsement.
