DevOps & Containerisation

Goal (≈10 hours)

Learn how to build an application with automated testing, containerise it and deploy it on a Kubernetes cluster. This training pack introduces you to modern DevOps practices through hands‑on tasks.

Suggested tools:

  • Git & GitHub
  • GitHub Actions or CI service
  • Docker
  • Minikube or kind (Kubernetes)
  • Terminal & Code editor
  • ChatGPT

Guided tasks

  1. Set up a CI pipeline. Create a simple application in your preferred language (e.g., Python, JavaScript). Host it in a Git repository and write a couple of unit tests. Configure a continuous integration workflow using GitHub Actions or another CI tool to run your tests on every push. Tip: Start with the official action templates to lint and test your project, and check the logs to debug failing steps.
  2. Containerise your application. Write a production‑grade Dockerfile that installs dependencies, copies your source code and defines an entrypoint. Build and run the container locally with docker build and docker run. Experiment with environment variables and mounted volumes to pass configuration. Try a multi‑stage build to reduce image size. Tip: Use docker ps and docker logs to inspect your running container.
  3. Deploy to a Kubernetes cluster. Use a local cluster such as Minikube or kind. Create a deployment and service YAML manifest to run your container with at least two replicas. Apply the configuration using kubectl apply -f, then verify with kubectl get pods and access your service via kubectl port-forward. Document the commands you used and note any errors encountered. Tip: Use official tutorials to understand deployments and services.
  4. Document & reflect. Write a short report describing each step: setting up CI, building the Docker image and deploying to Kubernetes. Include screenshots of your CI runs, Docker commands and Kubernetes resources. Reflect on the challenges you faced and how automation improves reliability. Consider how you would extend the pipeline to continuous deployment. Tip: ChatGPT can help you troubleshoot error messages and suggest improvements.

Tip: Use available tutorials, sample projects or ask ChatGPT for guidance as you complete each task.

Need help getting started? Break each task down into small steps, explore free tutorials online and ask ChatGPT to explain unfamiliar concepts. Practise iteratively and document your progress as you go.

What to submit

  • A link to your Git repository containing the application and CI configuration
  • Your Dockerfile and any related scripts
  • Kubernetes YAML manifests for your deployment and service
  • A concise report with screenshots and reflections on the process
